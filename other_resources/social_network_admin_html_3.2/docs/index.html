Package Overview

This template is built with Bootstrap 3.2.0 and provides multiple themes, layouts and skins. The package is structured in two branches:

  • dist - includes the static end product (HTML, images, compiled CSS and JavaScript)
  • dev - includes a full development suite based on the latest tools in front-end development, like Grunt, Jekyll, LESS, SASS and even a dedicated Vagrant environment so you can be developing in minutes with zero effort.

Development Branch Structure

│   │   ├── default.js          <!-- the default task to run when running grunt without specifying a task -->
│   │   ├── discover.js         <!-- special task for automatically generating a scripts vendor bundle from all the installed bower components -->
│   │   ├── jekyll.js           <!-- compiles all jekyll files to html -->

│   │   ├── less.js             <!-- compile LESS to CSS -->
│   │   ├── sass.js             <!-- compile SASS to CSS -->
│   │   ├── shell.js            <!-- execute shell commands -->

│   │   └── watch.js            <!-- Run predefined tasks whenever watched file patterns are added, changed or deleted -->

To make use of the development suite, depending on the specific functionality you'd like, you need to have installed node with npm, ruby, bower, grunt, gem, bundler and jekyll.

Easy alternative: If you'd like to experiment with the development suite but not bother with installing and configuring all these various tools, you can use the included Vagrant Environment - a dedicated virtual environment with everything you need pre-installed. You'll get a fully isolated virtual machine and be up & running in minutes. See more about how to use Vagrant on the development suite documentation.

JavaScript

Application scripts

All the various custom application scripts are bundled in a single file at dist/<theme_name>/js/scripts.js & also into a minified version scripts.min.js within the same directory.

Vendor scripts

All the vendor and 3rd party javascript files are bundled into a single file at dist<theme>/js/vendor.js & also into a minified version vendor.min.js within the same directory.

Styling

This amazing theme supports CSS pre-processing, not just from one, but two of the top industry-approved and mass-adopted CSS pre-processors: Less and Sass (CSS with superpowers) .

All the styling source is located within dev/app/less or dev/app/scss directories and are broken down in the following:
